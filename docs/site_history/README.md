# Site History

Archived context and retired plans. Everything here is historical: it never
overrides current state. Do not read it at session startup by default — use it
only for history, audit, or when a current memory file points to a specific
entry.

Move retired `docs/plans/` entries here once they are no longer resumable, and
promote any still-current rules into the relevant `docs/memory/<topic>.md` file
before archiving.
